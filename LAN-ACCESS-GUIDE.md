# 局域网访问指南 - 给员工用手机访问

## 方式一:局域网内访问(推荐,免费)

**适用场景**:员工和你连同一个 WiFi 或同一个路由器

### 步骤

#### 1. 启动服务器
双击运行 `start-lan.bat` 文件,会显示你的 IP 地址,例如:
- 本机 IP: `192.168.1.100`
- 局域网访问地址: `http://192.168.1.100:5173`

#### 2. 员工手机访问
1. 确保手机连接到和你电脑同一个 WiFi
2. 在手机浏览器中输入:`http://192.168.1.100:5173`(替换成你实际的 IP)
3. 可以正常浏览、搜索文章了!

#### 3. 配置防火墙(如果访问不了)
Windows 防火墙可能会阻止:

**方法 A:临时关闭防火墙测试**
- 控制面板 → Windows Defender 防火墙 → 启用或关闭 Windows Defender 防火墙
- 关闭专用网络和公用网络的防火墙
- 如果能访问了,说明是防火墙问题

**方法 B:添加允许规则(更安全)**
以管理员身份运行 PowerShell,执行:
```powershell
New-NetFirewallRule -DisplayName "Node.js HTTP Server" -Direction Inbound -LocalPort 5173,3001 -Protocol TCP -Action Allow
```

### 优缺点
- ✅ 完全免费
- ✅ 速度快,不经过外网
- ✅ 数据在自己电脑上,安全
- ❌ 必须保持电脑开机
- ❌ 必须在同一个局域网内(WiFi/路由器)
- ❌ 离开公司/家里就无法访问

## 方式二:内网穿透(让员工随时随地访问)

如果你想让员工在外面(比如在家、在路上)也能访问,需要用内网穿透工具。

### 免费方案:cpolar(推荐,限制访问速度)

#### 1. 注册账号
访问:https://www.cpolar.com/
注册免费账号

#### 2. 下载安装
下载 Windows 版本:https://dashboard.cpolar.com/downloads
安装后,在命令行执行:
```bash
cpolar authtoken 你的token
```

#### 3. 启动隧道
```bash
# 映射前端(5173端口)
cpolar http 5173

# 另开一个窗口,映射后端(3001端口)
cpolar http 3001
```

会给你一个公网地址,例如:
- `https://abc123.cpolar.cn`
- `https://xyz789.cpolar.cn`

#### 4. 修改前端配置
编辑 `blog/.env`,把后端地址改成内网穿透地址:
```
VITE_API_BASE_URL=https://xyz789.cpolar.cn
```

#### 5. 重启前端服务
关闭前端黑框框,重新运行 `npm run dev`

### 免费方案:ngrok(国外,速度慢)

1. 注册: https://ngrok.com/
2. 下载安装
3. 运行:
```bash
ngrok http 5173
```

### 付费方案(稳定,速度快)
- 花生壳(国内): https://hsk.oray.com/
- Frp 自建服务器(需要你有公网 IP)

### 优缺点
- ✅ 员工随时随地可以访问
- ✅ 不需要固定 WiFi
- ❌ 免费版本速度慢,有限制
- ❌ 付费需要花钱(几十元/月)
- ❌ 数据经过外网,需要注意安全

## 推荐方案总结

### 场景 1:员工在公司/家里,和你同一个 WiFi
→ **用方式一(局域网访问)**,免费又快

### 场景 2:员工需要在外面(如在家)也能访问
→ **用方式二(cpolar 免费版)**,虽然速度慢点但能用

### 场景 3:需要稳定、速度快,员工频繁使用
→ **用方式二(花生壳付费)**,几十元一个月,稳定可靠

## 常见问题

### Q1: 手机访问显示"无法连接"
**A**:
1. 检查手机是否连对 WiFi(和电脑同一个)
2. 检查 IP 地址是否正确
3. 尝试关闭 Windows 防火墙测试
4. 检查电脑是否开机,服务是否运行

### Q2: 访问很慢
**A**:局域网访问应该很快(几秒加载),如果很慢可能是:
- 网络问题
- 用了内网穿透(正常,免费版速度慢)
- 电脑性能太差

### Q3: 换了 WiFi 后访问不了
**A**:
- 局域网访问:IP 地址会变,需要重新运行 `start-lan.bat` 查看新 IP
- 内网穿透:地址不变,继续用原来的地址访问

### Q4: 需要给员工登录账号密码吗?
**A**:目前你的博客没有登录功能,所有人都能访问。如果需要登录功能,我可以帮你加上。

### Q5: 数据安全吗?
**A**:
- 局域网访问:数据在你自己电脑上,很安全
- 内网穿透:数据经过外网,但 HTTPS 加密了,基本安全
- 如果需要更高的安全性,可以考虑添加登录功能和 HTTPS 证书
