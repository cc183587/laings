## 2026-03-23 第六次更新 - 批次利润管理

### 新增功能
1. **批次利润管理模块**：在管理员面板新增「批次利润」标签页
2. **创建批次（入库）**：
   - 选择产品（从产品列表选择）
   - 输入入库数量
   - 输入单价（元/件）
   - 自动计算入库金额 = 数量 × 单价
   - 批次ID自动生成（格式：B + 时间戳）

3. **出库结算**：
   - 点击「出库结算」按钮完成批次结算
   - 自动计算利润 = 入库金额 - 员工工资总额
   - 显示确认对话框，包含入库金额、员工工资、利润明细
   - 结算后批次状态改为「已出库」

4. **批次列表显示**：
   - 批次ID、产品名称、状态（入库中/已出库）
   - 入库数量、单价、入库金额
   - 员工工资总额（自动统计）
   - 预估利润（入库中时）/ 实际利润（已出库时）

5. **导出批次利润报表**：导出包含批次ID、产品、入库金额、员工工资、利润等信息的Excel报表

### 数据结构
新增 `data.batches` 数组，存储批次信息：
- id: 批次ID
- prodKey: 产品key
- prodName: 产品名称
- qty: 入库数量
- price: 单价
- totalCost: 入库金额
- createTime: 创建时间
- status: 状态（in/out）
- wageTotal: 员工工资总额（简化为固定计算，不再动态关联）
- profit: 利润

### 使用流程
1. 管理员创建批次（选择产品、输入数量和单价）
2. 员工正常登记产量（无需选择批次）
3. 员工完成所有产量后，管理员点击「出库结算」
4. 系统显示利润明细，管理员确认后结算完成

### 使用场景

4. 系统自动计算：
   - 实际出库金额：490 × 0.8 = 392元
   - 坏品损失：400 - 392 = 8元
   - 利润：392 - 员工工资总额

2. **改写所有受影响函数为 async**：
   - `closeBatch()` → 出库数量输入 + 结算确认
   - `delBatch()` → 删除确认（红色危险按钮）
   - `adminDelEmp()` → 删除确认
   - `adminDelProduct()` → 删除确认
   - `adminDelOneRecord()` → 删除确认
